# Guía de Contribución 🤝

¡Gracias por tu interés en contribuir al sitio web de **AWS Student Builder Group - Antonio Varas**!

Como somos una comunidad de estudiantes, valoramos cualquier tipo de aporte, desde arreglar un pequeño error tipográfico hasta crear componentes nuevos enteros.

## ¿Cómo empezar?

1. Haz un **Fork** de este repositorio.
2. Clona el repositorio en tu máquina local: `git clone https://github.com/TU-USUARIO/Web-SBG.git`
3. Crea una rama para tu feature o bugfix: `git checkout -b feature/mi-nueva-idea` o `git checkout -b fix/error-formulario`.
4. Realiza tus cambios y asegúrate de probarlos localmente con `npm run dev`.
5. Revisa que tu código cumpla con los estándares ejecutando `npm run lint` y `npm run format`.
6. Haz commit de tus cambios: `git commit -m "feat: agrego nueva sección de eventos"`.
7. Haz push a tu fork: `git push origin feature/mi-nueva-idea`.
8. Abre un **Pull Request** (PR) hacia la rama `main` de este repositorio.

## Estándares de Código

Usamos **ESLint** y **Prettier** para mantener el código uniforme. Tu Pull Request debe pasar el flujo de integración continua (CI) en GitHub Actions.

## Reporte de Errores (Bugs) y Propuestas (Features)

- Si encuentras un error, por favor abre un **Issue** usando la plantilla de Bug Report.
- Si tienes una idea genial, abre un **Issue** usando la plantilla de Feature Request.

¡Gracias por hacer crecer esta comunidad!
